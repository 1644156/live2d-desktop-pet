# Live2D Desktop Pet 功能扩展建议汇总

## 1. 项目现状判断

基于当前代码结构与 README，可以确认这个项目已经不再是“会说话的桌宠 Demo”，而是一个具备 Agent 雏形的桌面 AI 伴侣系统，已具备以下基础能力：

- Live2D 桌面渲染与鼠标跟随
- 多窗口 UI（主窗口、输入框、历史记录、待办、知识库）
- 基于 LangGraph 的对话流程编排
- 多模型适配与工具调用
- 长期记忆、情绪存储、对话持久化
- 本地知识库 / RAG
- 提醒事项与时间解析
- TTS 语音输出
- Persona / 角色设定

这意味着后续扩展不应只做“再加一个工具”，而应重点提升三件事：

1. **角色真实感**：让桌宠更像“活着的角色”而不是 UI 外壳。
2. **主动性与陪伴感**：从被动问答升级到持续陪伴。
3. **桌面级 Agent 能力**：真正融入用户的电脑使用场景。

---

## 2. 总体扩展原则

建议后续功能设计遵循以下原则：

- **优先增强已有长板**：优先放大记忆、情绪、角色、桌面交互这几个已有优势。
- **避免纯堆功能**：新增功能要能反哺角色感、主动性或生产力，不建议做孤立功能点。
- **本地优先、隐私优先**：涉及截图、语音、记忆、文件操作时，优先提供本地化方案。
- **低打扰、高存在感**：桌宠应始终“在”，但不能频繁打断用户工作。
- **表现层和能力层同步升级**：新增能力时，同时考虑 Live2D 动作、表情、语音和 UI 表现。

---

## 3. 全量扩展建议

### 3.1 Live2D 表现力升级

#### 1) 触碰反馈系统
利用 Live2D 的 HitArea 或前端命中检测，实现不同身体区域的交互反馈。

可做的效果：
- 摸头：开心、害羞、安抚型回复
- 戳脸：不满、吐槽、闪躲动作
- 点击手部：挥手、招呼、请求互动
- 长按角色：触发特殊台词或亲密动作

价值：
- 直接提升“角色活着”的感觉
- 明显增强非文本交互体验

#### 2) 更完整的待机行为系统
当前角色可继续向“持续存在”演进。

建议加入：
- 随机待机动作
- 长时间无交互时的发呆、打哈欠、看向别处
- 不同情绪状态下不同待机动画池
- 凌晨 / 深夜 / 雨天 / 节日专属 idle 行为

价值：
- 用户不输入时，桌宠也不会显得像“暂停的程序”

#### 3) 口型同步与呼吸同步
将 TTS 播放状态和音频能量映射到 Live2D 参数。

建议实现：
- `ParamMouthOpenY` 与音频音量包络绑定
- 呼吸参数缓慢波动
- 说话时头部轻微摆动

价值：
- 会大幅提升语音输出时的沉浸感

#### 4) 视线与头部跟随增强
你已经有鼠标跟随基础，可以进一步细化：

- 眼球单独跟随鼠标
- 头部角度缓动而不是瞬移
- 鼠标停留在角色周边时触发“注视你”的状态
- 打字时看向输入框，播放语音时看向气泡

---

### 3.2 主动陪伴与事件驱动系统

#### 5) 主动发起对话
桌宠从“等你说话”升级为“会找你说话”。

触发条件可以包括：
- 长时间无输入
- 深夜仍在使用电脑
- 连续工作超过 1~2 小时
- 今天心情持续偏低
- 某个提醒事项即将到来

示例：
- “今天你已经坐很久了，要不要起来活动一下？”
- “你今天好像一直很忙，我就安静陪着你。”

关键点：
- 提供“主动程度”配置
- 加入冷却时间，防止过度打扰

#### 6) 事件聚合提醒中心
除了待办提醒，可以进一步聚合更多事件源：

- 新邮件
- 日历日程
- 系统通知
- 天气突变
- 电量过低
- 网络断开
- 长时间高 CPU / 高温

桌宠可以：
- 通过气泡提示
- 用动作和表情表达事件紧急程度
- 在重要事件时进行 TTS 播报

#### 7) 生活节律陪伴
让桌宠具备轻量生活助理能力：

- 喝水提醒
- 睡眠提醒
- 久坐提醒
- 番茄钟陪伴
- 学习 / 工作模式切换

建议结合角色台词风格，而不是纯工具式提醒。

---

### 3.3 语音闭环升级

#### 8) 加入本地 ASR（语音识别）
目前已有 TTS，下一步最自然的是补齐语音输入闭环。

建议：
- 接入 Faster-Whisper 或 Whisper.cpp
- 支持按住说话 / 唤醒词 / 自动端点检测（VAD）
- 支持中日双语识别

价值：
- 从“桌面宠物”升级为“桌面语音助手”

#### 9) 唤醒词与连续对话模式
可加入类似：
- “安安” / “小助手” 唤醒
- 唤醒后进入几秒连续监听窗口
- 连续对话结束后自动退回安静模式

#### 10) 更高级的音色系统
未来可以支持：
- 多角色音色配置
- 不同情绪使用不同语气 / 语速 / 音高
- 本地 TTS 备选方案
- 多个角色共存时分别拥有不同声线

---

### 3.4 Agent 工具体系升级

#### 11) 本地系统控制工具
目前已有天气、时间、待办、知识库，下一步最有价值的是接入本地系统控制。

建议工具：
- 打开应用
- 打开文件夹
- 调整音量
- 调整亮度
- 锁屏
- 关闭屏幕
- 切换电源模式
- 调起常用网址

注意：
- 需要权限白名单
- 危险操作需二次确认

#### 12) 剪贴板与选中文本助手
非常适合桌面场景。

建议功能：
- 读取当前剪贴板文本
- 快捷键解释选中的代码 / 文本
- 一键改写、翻译、总结
- 把选中文本写入知识库或记忆

价值：
- 实用性极高，开发成本相对可控

#### 13) 文件整理与自动化助手
可以逐步发展为轻量桌面 RPA：

示例：
- 把下载目录中的图片整理到日期文件夹
- 把桌面的 PDF 分类归档
- 把会议纪要自动汇总到指定目录

注意：
- 先从“建议模式”开始，再开放“自动执行模式”

#### 14) 终端 / 开发辅助工具
如果项目面向开发者用户，这一项很有价值：

- 运行受限白名单命令
- 解析错误日志
- 帮用户解释构建失败原因
- 快速生成 commit message / release note

---

### 3.5 知识库与 RAG 继续增强

#### 15) 联网搜索回退
本地知识库查不到时，自动回退到联网搜索。

建议流程：
1. 先查记忆
2. 再查知识库
3. 结果不足时询问是否联网搜索
4. 返回结果后可选择写回知识库

这样可以兼顾：
- 本地优先
- 时效性问题
- 用户可控性

#### 16) 知识库结构化管理
当前知识库可继续增强为“可管理的信息空间”：

建议加入：
- 标签 / 分类
- 文档来源标记
- 导入时间 / 更新时间
- 文档摘要自动生成
- 文档去重与重复提醒
- 手动禁用某些文档参与检索

#### 17) 长文档阅读模式
面对大型 PDF / 规范文档时，可提供：

- 分章节摘要
- 针对某一页 / 某一章提问
- 自动生成术语表
- 对文档进行问答缓存

#### 18) 从对话反向沉淀知识
把有价值对话转成长期知识资产：

- 自动识别“值得记录”的问答
- 一键保存当前对话为知识条目
- 将历史高频问题汇总成 FAQ

---

### 3.6 记忆、关系与人格成长

#### 19) 关系值 / 好感度系统
情绪系统已经有基础，可以继续扩展为长期关系模型。

可记录的维度：
- 亲密度
- 信任度
- 最近互动频率
- 最近情绪质量
- 用户对角色的态度变化

这些值可以影响：
- 称呼方式
- 回复距离感
- 主动程度
- 可解锁台词 / 动作 / 特殊剧情

#### 20) Persona 动态演化
目前 `persona.py` 是静态配置为主，可扩展成“成长型角色”。

可演化内容：
- 说话风格逐渐贴近用户偏好
- 根据长期对话发展出新的习惯表达
- 对特定话题形成稳定偏好
- 根据用户反馈收敛角色边界

#### 21) 记忆分层
建议将记忆进一步分层，而不只是简单向量召回：

- 短期上下文记忆
- 中期话题记忆
- 长期人物画像
- 高优先级核心设定记忆
- 禁止遗忘的锚点记忆

这样可以降低“记错设定”或“重要内容被噪声稀释”的概率。

#### 22) 特殊纪念日与长期陪伴节点
让桌宠拥有“时间感”：

- 记住第一次见面日期
- 记住生日、重要节点
- 每周 / 每月做小结
- 节日时切换专属问候和主题

---

### 3.7 多模态能力扩展

#### 23) 截图问答
提供快捷键截图，然后让桌宠分析：

- 代码报错截图
- 网页内容
- PPT / 文档页面
- 游戏画面
- UI 设计稿

可接入多模态模型，产出：
- 解释
- 总结
- 改写
- 翻译
- 操作建议

#### 24) OCR 与屏幕上下文理解
比单次截图更进一步：

- 读取当前活动窗口标题
- OCR 当前屏幕文本
- 对屏幕场景进行摘要
- 给出与当前工作场景相关的帮助

#### 25) 摄像头感知（可选、默认关闭）
这是偏实验型但很有亮点：

- 追踪用户头部方向
- 检测是否离开座位
- 根据表情判断疲劳或情绪

注意：
- 必须默认关闭
- 必须明确隐私提示

---

### 3.8 生产力与日常工作流

#### 26) 日历 / 邮件 / IM 联动
如果后续引入 MCP 或第三方连接器，这一项很强：

- 今日日程摘要
- 会前提醒
- 邮件摘要
- 待回复事项归纳
- 消息聚合看板

#### 27) 学习 / 写作 / 编码模式
为不同使用场景提供不同工作流：

- 学习模式：总结、抽认卡、术语解释
- 写作模式：润色、扩写、改写、提纲
- 编码模式：错误分析、代码解释、文档生成
- 会议模式：录音转纪要、待办提取

#### 28) 番茄钟和专注陪伴
这是桌宠形态非常适合做的一项：

- 开始专注时角色进入安静模式
- 休息时恢复活跃并提醒放松
- 完成一个番茄后给予反馈和鼓励
- 和好感度系统联动，形成养成感

---

### 3.9 角色生态与内容系统

#### 29) 多角色切换
目前已有 persona 基础，非常适合扩展：

- 不同 Live2D 模型绑定不同 persona
- 不同角色拥有独立记忆或共享公共记忆
- 切换角色时保留各自情绪和关系状态

#### 30) 角色技能差异化
不同角色不只是皮肤不同，而是能力不同：

- 学习型角色擅长总结
- 开发型角色擅长代码解释
- 治愈型角色擅长陪伴与情绪回应
- 管家型角色擅长提醒和日程管理

#### 31) 角色剧情 / 收藏 / 解锁系统
如果想增强产品粘性，可以做轻量养成：

- 特殊事件解锁台词
- 好感度阶段剧情
- 节日限定动作 / 服装 / 问候
- 成就系统（连续打卡、连续专注、完成任务数）

---

### 3.10 工程化与产品化能力

#### 32) 插件化工具系统
随着工具变多，建议将 `src/chat/tools/` 进一步插件化：

- 统一工具注册协议
- 工具权限分级
- 工具描述标准化
- UI 中动态显示可用工具

这样后续接入更多工具时维护成本会更低。

#### 33) 设置中心
项目功能越来越多后，需要一个统一设置中心：

建议包含：
- 主动打扰程度
- TTS/ASR 开关
- 权限控制
- 记忆策略
- 知识库策略
- 外观主题
- 性能模式

#### 34) 调试面板 / 开发者模式
对后续迭代很有帮助：

- 查看当前 prompt
- 查看召回到的记忆
- 查看知识库检索结果
- 查看情绪分析结果
- 查看工具调用链路

#### 35) 资源模式与性能模式
桌宠产品很需要考虑资源占用：

- 低功耗模式
- 静音模式
- 工作模式 / 游戏模式
- GPU / CPU 占用限制
- 后台空闲时降低刷新率

#### 36) 自动更新与配置备份
产品化阶段应考虑：

- 配置导出 / 导入
- 用户数据备份
- 角色数据迁移
- 自动更新机制

## 4. 新增补充建议（前两轮之外值得考虑的点）

### 4.1 回忆触发器
让角色不仅“记得”，还会“想起”。

示例：
- 用户再次提到某个旧项目时主动提及上次讨论结果
- 节假日时想起去年的互动
- 在相似情绪下回忆过去的对话

这比单纯向量召回更像真实陪伴。

### 4.2 双层回复机制
可以把回复拆成两层：

- 外层：简短角色化即时回应（气泡）
- 内层：展开回答 / 工具结果 / 深度解释（历史窗口或面板）

这样既能保持桌宠轻盈感，又能承载复杂能力。

### 4.3 行为一致性系统
让同一角色在不同模块中保持一致：

- 提醒语气一致
- 错误提示也保持角色语气
- 知识库总结、待办创建、天气播报都符合人设

### 4.4 轻量状态可视化
把内部状态适度可视化给用户：

- 当前心情
- 当前忙碌状态
- 是否正在检索知识库
- 是否正在记忆写入
- 今日互动次数 / 好感度变化

这会显著增强“角色存在感”。

## 6. 推荐实施路线

### 第一阶段：把“陪伴感”做扎实
目标：让桌宠更像角色，而不是聊天工具。

推荐先做：
- 触碰反馈
- 待机行为
- 主动关怀
- 好感度 / 关系值
- 角色一致性台词体系

### 第二阶段：把“桌面助手”做实用
目标：让用户愿意每天用。

推荐做：
- 本地 ASR
- 剪贴板助手
- 联网搜索回退
- 截图问答
- 日程 / 番茄钟 / 专注模式

### 第三阶段：把“Agent 化”做完整
目标：从桌宠升级为桌面 AI Companion Agent。

推荐做：
- 系统控制工具
- 插件化工具体系
- 多角色生态
- 跨设备同步
- 自动化工作流

## 7. 落地时需要特别注意的问题

### 安全与权限
- 文件操作、系统控制、截图、摄像头、麦克风都必须有明确权限边界
- 危险动作应二次确认
- 默认保守，逐步放开

### 性能与资源占用
- Live2D + WebView + 向量检索 + TTS/ASR 容易叠加占用
- 要尽早规划性能模式、懒加载和缓存策略

### 记忆污染
- 长期记忆应允许用户查看、删除、纠正
- 记忆提取要区分事实、偏好、猜测、临时情绪

### 角色一致性
- 新功能加入时，要同步考虑 persona、台词、动画、声音的统一体验
- 不然容易变成“强功能、弱角色”的割裂产品

### 打包与分发
- 引入 ASR、多模态、更多本地模型后，安装包体积会明显膨胀
- 建议规划“基础版 + 增强包”结构

## 8. 我最推荐的组合方案

如果要选择一条最适合当前项目的升级路线，我最推荐下面这组组合：

### 组合 A：角色感优先
- 触碰反馈
- 待机行为
- 好感度系统
- 主动关怀
- 口型同步

适合目标：把项目做成真正有“陪伴感”的桌宠产品。

### 组合 B：实用性优先
- 本地 ASR
- 剪贴板助手
- 联网搜索回退
- 截图问答
- 日程 / 番茄钟模式

适合目标：让项目成为高频桌面助手。

### 组合 C：Agent 化优先
- 系统控制工具
- 插件化工具体系
- 知识库结构化管理
- 多角色技能差异化
- 调试面板

适合目标：把项目做成可扩展的桌面 AI Agent 平台。

## 9. 结论

这个项目最有价值的方向，不是单纯继续加工具，而是围绕以下三条主线持续强化：

1. **更像角色**：触碰、动作、语音、情绪、关系成长
2. **更像陪伴者**：主动关怀、时间感、记忆触发、低打扰持续存在
3. **更像桌面 Agent**：剪贴板、截图、系统工具、知识库、自动化工作流

如果只选一个方向先做，我建议优先做：

- **触碰反馈 + 待机行为 + 主动关怀 + 好感度系统**

因为这组改动最能体现你这个项目区别于普通 AI 聊天壳子的核心竞争力。
