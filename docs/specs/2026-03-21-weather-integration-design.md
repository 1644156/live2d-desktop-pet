# 天气查询工具集成设计

## 1. 概述

本文档描述了为 Live2D 桌宠系统集成天气查询功能的设计方案。基于 `weather.md` 提供的 API 规范，使用 LangGraph 框架实现天气数据的获取、处理和响应。

## 2. 目标

- 实现天气查询功能，支持自动定位和城市指定
- 支持查询今天及未来天气预报
- 根据天气情况返回贴心建议
- 保持现有对话架构，最小化改动
- 便于后续扩展更多工具

## 6. 天气工具设计

### 6.1 工具定义

```python
@tool
def get_weather(city: str = None, date: str = None) -> str:
    """
    查询指定城市和日期的天气信息。

    当用户询问以下内容时应该调用此工具：
    - 询问天气情况（如"今天天气怎么样"、"天气如何"）
    - 询问温度（如"今天多少度"、"热不热"）
    - 询问未来天气（如"明天天气如何"、"后天会下雨吗"）
    - 询问特定日期天气（如"3月23日的天气"、"周末天气怎么样"）
    - 询问是否需要带伞、穿什么衣服等与天气相关的问题
    - 提到具体城市名称并询问天气（如"北京天气"、"上海明天热吗"）

    参数:
        city: 城市名称（可选），如"北京"、"上海"、"广州"。
              如果不提供，则自动通过IP定位获取当前位置。
        date: 查询日期（可选），支持以下格式：
              - "今天" 或 不填：查询当天天气
              - "明天"：查询明天天气
              - "后天"：查询后天天气
              - 具体日期："3月23日"、"2024-03-23"
              最多支持未来7天的天气预报。

    返回:
        包含城市名称、日期、天气状况、温度以及贴心建议的友好回复。
    """
```

### 6.2 IP定位实现

使用太平洋网络 IP 定位 API：

```python
def get_current_location() -> dict:
    """通过IP获取当前位置（使用太平洋网络API）"""
    try:
        response = requests.get(
            "https://whois.pconline.com.cn/ipJson.jsp?json=true",
            timeout=5
        )
        data = response.json()
        city = data.get("city", "").replace("市", "")
        province = data.get("pro", "").replace("省", "")
        return {
            "city": city if city else province,
            "province": province,
            "ip": data.get("ip", "")
        }
    except Exception:
        return {"city": "北京", "province": "北京", "ip": ""}
```

def get_weather_suggestion(weather_code: int, temp: float) -> str:
    """根据天气代码和温度返回建议"""
    import random
    
    # 优先根据天气代码判断
    if weather_code in [0, 1]:
        category = "sunny"
    elif weather_code in [2, 3]:
        category = "cloudy"
    elif weather_code in [45, 48]:
        category = "fog"
    elif weather_code in [51, 53, 55, 56, 57]:
        category = "light_rain"
    elif weather_code in [61, 63, 65, 66, 67]:
        category = "rain"
    elif weather_code in [80, 81, 82]:
        category = "heavy_rain"
    elif weather_code in [71, 73]:
        category = "light_snow"
    elif weather_code in [75, 77]:
        category = "snow"
    elif weather_code in [95, 96, 99]:
        category = "thunderstorm"
    else:
        category = "cloudy"
    
    # 温度覆盖建议
    if temp >= 35:
        category = "hot"
    elif temp <= 0:
        category = "freezing"
    elif temp <= 5:
        category = "cold"
    
    suggestions = WEATHER_SUGGESTIONS.get(category, WEATHER_SUGGESTIONS["cloudy"])
    return random.choice(suggestions)

### 6.4 错误处理

| 异常场景 | 处理方式 |
|---------|---------|
| 城市名无法识别 | 返回："抱歉，找不到城市'{city}'呢" |
| 网络超时 | 返回："网络不太好，稍后再试试～" |
| API服务异常 | 返回："天气服务暂时不可用呢～" |
| 定位失败 | 默认使用北京作为备选位置 |
