# Live2D 口型同步改造方案

## 一、现状分析

### 1.1 当前实现

现在的口型同步是**假的**。整条链路如下：

```
TTS 合成完一句话
-> Python 拿到总时长 duration
-> WebBridge.start_lip_sync(duration)
-> JS startLipSync(durationSec)
-> 用 Math.sin() 生成固定频率的正弦曲线
-> 赋给 motionManager.mouthSync
-> 嘴巴按固定节奏机械开合
```

**涉及文件和行号：**

| 环节 | 文件 | 行号 | 做了什么 |
|------|------|------|----------|
| 触发口型 | `src/app/chat_manager.py` | 770-772 | 拿到 duration，调 `start_lip_sync(duration)` |
| 桥接调用 | `src/app/web_bridge.py` | 81-84 | `runJavaScript("startLipSync(duration)")` |
| 假口型生成 | `public/desktop.html` | 274-287 | 3 个 sin() 叠加，生成 0~1 的假振幅 |
| 停止口型 | `public/desktop.html` | 290-306 | 清除假音频对象，恢复原始 mouthSync |

**核心问题：**

- 嘴巴运动和实际音频内容完全无关
- 不管说什么，嘴巴都是同一个节奏
- 停顿时嘴巴仍在动
- 无法区分不同元音的口型

### 1.2 TTS 音频流现状

TTS 使用阿里 DashScope 的 WebSocket 实时合成接口，**已经是流式的**：

| 参数 | 值 | 代码位置 |
|------|------|------|
| 采样率 | 24000 Hz | `tts_service.py:191`, `audio_player.py:11` |
| 位深 | 16-bit signed int | `audio_player.py:93` |
| 声道 | 单声道 | `audio_player.py:12` |
| 每秒字节数 | 48000 bytes/s | `audio_player.py:14` |
| 播放缓冲区 | 1024 frames | `audio_player.py:97` |

**关键截获点：**

- `tts_service.py:165-169`：收到 `response.audio.delta`，Base64 编码的 PCM 增量数据
- `audio_player.py:107-108`：`_play_loop` 中 `self._stream.write(audio_data)`，此处是即将送入声卡的原始 PCM

### 1.3 模型口型参数现状

对项目中全部 9 个模型的 `.model3.json` 进行了扫描：

| 模型 | LipSync 绑定 | ParamMouthOpenY | ParamMouthForm | ParamMouthA/I/U/E/O | 其他特殊参数 |
|------|-------------|-----------------|----------------|---------------------|-------------|
| Mao | ParamA | 无 | ParamMouthForm | 无 | ParamA 即嘴张开 |
| Nahida | ParamMouthOpenY | 有 | ParamMouthForm | 无 | - |
| 爱莉希雅 | 空 | 有 | ParamMouthForm | 无 | 吐舌、歪嘴、鼓脸 |
| 藿藿 | 空 | 有 | ParamMouthForm | 无 | 歪嘴 |
| yumi | 空 | 有 | ParamMouthForm | 无 | JawOpen、MouthX、MouthShrug、Cheekpuff |
| Mandou | 空 | 有 | ParamMouthForm | 无 | - |
| 90_A | 空 | 有 | ParamMouthForm | 无 | - |
| 90_B | 空 | 有 | ParamMouthForm | 无 | - |
| UG | 空 | 有 | ParamMouthForm | 无 | - |

**结论：**

- **没有任何模型支持 ParamMouthA/I/U/E/O**，所以"多元音口型"在当前模型上无法实现
- **所有模型都有 ParamMouthOpenY**（或等价的 ParamA），可以控制嘴巴开合
- **大部分模型有 ParamMouthForm**，可以控制嘴型形状（正值偏笑/横向拉伸，负值偏圆/嘟嘴）
- 部分模型的 LipSync 分组 Ids 为空，需要代码主动查找可用参数

## 二、改造目标

### 能做到的（本方案范围）

1. **嘴巴跟着实际声音动**：有声音时张嘴，停顿时闭嘴，声音大张得大，声音小张得小
2. **区分发音特征**：通过音频频率特征，粗略区分"扁嘴音"和"圆嘴音"，驱动 `ParamMouthForm`
3. **平滑自然**：快开慢闭，不抽搐不跳变
4. **自动适配模型**：检测模型支持哪些参数，自动选择最佳驱动模式
5. **不引入外部工具**：纯 Python 标准库 + 已有依赖，不需要 Rhubarb、OVRLipsync 等

### 做不到的（明确边界）

- 精确的音素级口型（需要外部强制对齐器）
- A/I/U/E/O 五元音独立驱动（需要模型支持对应参数）
- 和配音软件一样精准的唇形匹配

## 三、技术方案

### 3.1 整体架构

改造后的数据流：

```
TTS WebSocket 收到 PCM 增量
-> AudioPlayer._play_loop 播放音频
-> 同时将 PCM chunk 送入 LipSyncAnalyzer

-> 通过信号发送到主线程
-> WebBridge.update_mouth_params(open_y, mouth_form)
-> JS 直接写 coreModel 参数
```

## 四、详细设计

### 4.1 新增：LipSyncAnalyzer 模块

**文件：** `src/tts/lipsync_analyzer.py`

**职责：** 接收原始 PCM 字节流，实时提取音频特征，输出口型参数。

#### 4.1.1 音频特征提取

从每个 PCM chunk 中提取三个特征，不依赖 numpy/scipy，纯 Python 标准库 + `array` 模块：

**特征 1：RMS（均方根响度）**

```python
import math
from array import array

def calc_rms(samples: array) -> float:
    """计算 PCM 样本的 RMS 响度，归一化到 0.0~1.0"""
    if not samples:
        return 0.0
    sq_sum = sum(s * s for s in samples)
    return math.sqrt(sq_sum / len(samples)) / 32768.0
```

用途：控制 `ParamMouthOpenY` 的目标值。RMS 越大，嘴张得越大。

**特征 2：ZCR（过零率）**

```python
def calc_zcr(samples: array) -> float:
    """计算过零率，范围 0.0~1.0"""
    if len(samples) < 2:
        return 0.0
    crossings = sum(
        1 for i in range(1, len(samples))
        if (samples[i-1] < 0) != (samples[i] < 0)
    )
    return crossings / (len(samples) - 1)
```

用途：区分发音类型。
- 高 ZCR (> 0.2)：摩擦音/清辅音（s, sh, f, x），嘴型偏扁/横拉
- 低 ZCR (< 0.1)：浊元音（a, o, u），嘴型偏圆/纵向

**特征 3：频带能量比（Goertzel 算法）**

不用 FFT，用 Goertzel 只算我们关心的几个频段的能量：

```python
def goertzel_energy(samples: array, target_freq: float, sample_rate: int = 24000) -> float:
    """用 Goertzel 算法计算单个频率的能量"""
    n = len(samples)
    if n == 0:
        return 0.0
    k = int(0.5 + n * target_freq / sample_rate)
    w = 2.0 * math.pi * k / n
    coeff = 2.0 * math.cos(w)
    s0 = 0.0
    s1 = 0.0
    s2 = 0.0
    for sample in samples:
        s0 = sample + coeff * s1 - s2
        s2 = s1
        s1 = s0
    return s1 * s1 + s2 * s2 - coeff * s1 * s2

def calc_band_ratio(samples: array, sample_rate: int = 24000) -> dict:
    """计算低/中/高频带的能量分布"""
    low = goertzel_energy(samples, 500, sample_rate)    # 低频 (~500Hz)
    mid = goertzel_energy(samples, 1200, sample_rate)   # 中频 (~1200Hz)
    high = goertzel_energy(samples, 2500, sample_rate)  # 高频 (~2500Hz)
    total = low + mid + high + 1e-10
    return {
        "low": low / total,
        "mid": mid / total,
        "high": high / total
    }
```

用途：辅助判断口型形状。
- 低频占比高：偏 A/O 类口型（张口、圆口）
- 高频占比高：偏 I/E/S 类口型（扁口、齿音）

#### 4.1.2 口型参数计算规则

将三个特征组合，输出两个 Live2D 参数的目标值：

```python
def compute_viseme(rms: float, zcr: float, bands: dict) -> tuple:
    """
    返回 (open_y, mouth_form)
    open_y:     0.0 (闭嘴) ~ 1.0 (大张嘴)
    mouth_form: -1.0 (圆嘴/嘟嘴) ~ 1.0 (扁嘴/笑)
    """
    # --- open_y：基于 RMS ---
    # 非线性映射，让小声音也能有明显张嘴
    if rms < 0.01:
        target_open = 0.0  # 静音，闭嘴
    else:
        # 用 sqrt 做非线性拉伸，让中等音量也有明显口型
        normalized = min(rms / 0.15, 1.0)   # 0.15 以上视为满幅
        target_open = math.sqrt(normalized)  # sqrt 拉伸
        target_open = max(0.08, target_open) # 有声音时最小张嘴 0.08

    # --- mouth_form：基于 ZCR + 频带 ---
    if rms < 0.01:
        target_form = 0.0  # 静音不改变形状
    elif zcr > 0.20:
        # 高过零率：摩擦音/清辅音 -> 扁嘴
        target_form = 0.4 + (zcr - 0.20) * 2.0  # 越高越扁
        target_form = min(target_form, 0.8)
    elif bands["low"] > 0.5:
        # 低频主导：O/U 类 -> 圆嘴
        target_form = -0.3 - (bands["low"] - 0.5) * 1.0
        target_form = max(target_form, -0.7)
    elif bands["high"] > 0.4:
        # 高频主导：I/E/S 类 -> 微扁
        target_form = 0.2 + (bands["high"] - 0.4) * 1.5
        target_form = min(target_form, 0.6)
    else:
        # 中频主导：A 类 -> 自然张口
        target_form = 0.0

    return (target_open, target_form)
```

#### 4.1.3 平滑策略：快开慢闭

```python
class LipSyncSmoother:
    """口型平滑器，模拟嘴部肌肉惯性"""

    def __init__(self):
        self.current_open = 0.0
        self.current_form = 0.0
        self.attack = 0.6   # 张嘴速度（越大越快）
        self.release = 0.15  # 闭嘴速度（越小越慢）
        self.form_speed = 0.25  # 形状变化速度

    def smooth(self, target_open: float, target_form: float) -> tuple:
        # open_y: 快开慢闭
        if target_open > self.current_open:
            speed = self.attack
        else:
            speed = self.release
        self.current_open += (target_open - self.current_open) * speed

        # mouth_form: 统一中速
        self.current_form += (target_form - self.current_form) * self.form_speed

        # 阈值截断，防止无限逼近
        if abs(self.current_open) < 0.01:
            self.current_open = 0.0
        if abs(self.current_form) < 0.01:
            self.current_form = 0.0

        return (self.current_open, self.current_form)
```

#### 4.1.4 完整类设计

```python
# src/tts/lipsync_analyzer.py

import math
import time
from array import array
from PyQt5.QtCore import QObject, pyqtSignal


class LipSyncAnalyzer(QObject):
    """
    实时音频口型分析器
    接收 PCM 字节流，输出 Live2D 口型参数
    """
    visemeUpdated = pyqtSignal(float, float)  # (open_y, mouth_form)

    SAMPLE_RATE = 24000
    MIN_INTERVAL = 0.025  # 最小更新间隔 25ms（约 40fps）

    def __init__(self, parent=None):
        super().__init__(parent)
        self._smoother = LipSyncSmoother()
        self._last_update_time = 0.0
        self._is_speaking = False

    def start(self):
        """开始说话"""
        self._is_speaking = True
        self._smoother.current_open = 0.0
        self._smoother.current_form = 0.0

    def stop(self):
        """停止说话，嘴巴回到闭合"""
        self._is_speaking = False
        self.visemeUpdated.emit(0.0, 0.0)

    def feed(self, pcm_bytes: bytes):
        """
        喂入一块 PCM 数据（16-bit signed, mono, 24kHz）
        在 AudioPlayer 的播放线程中调用
        """
        if not self._is_speaking:
            return

        now = time.time()
        if now - self._last_update_time < self.MIN_INTERVAL:
            return
        self._last_update_time = now

        # 解析 PCM -> int16 数组
        samples = array("h")
        samples.frombytes(pcm_bytes)
        if len(samples) < 64:
            return

        # 提取特征
        rms = calc_rms(samples)
        zcr = calc_zcr(samples)
        bands = calc_band_ratio(samples, self.SAMPLE_RATE)

        # 计算目标口型
        target_open, target_form = compute_viseme(rms, zcr, bands)

        # 平滑
        smooth_open, smooth_form = self._smoother.smooth(target_open, target_form)

        # 发送信号
        self.visemeUpdated.emit(smooth_open, smooth_form)
```

#### 4.4.1 新增：模型口型能力检测

在模型加载完成后（`loadModel` 函数末尾），自动检测嘴部参数：

```javascript
// 口型能力检测结果（全局）
let mouthCapability = {
    mode: 'unsupported',  // 'open_form' | 'open_only' | 'unsupported'
    openParam: null,      // 实际可用的张嘴参数名
    formParam: null       // 实际可用的嘴型参数名
};

function detectMouthCapabilities() {
    mouthCapability = { mode: 'unsupported', openParam: null, formParam: null };

    if (!model || !model.internalModel || !model.internalModel.coreModel) {
        return mouthCapability;
    }

    const coreModel = model.internalModel.coreModel;

    // 候选参数名（按优先级排列）
    const openCandidates = [
        'ParamMouthOpenY',
        'ParamA',
        'PARAM_MOUTH_OPEN_Y',
        'MouthOpenY',
        'MouthOpen'
    ];

    const formCandidates = [
        'ParamMouthForm',
        'PARAM_MOUTH_FORM',
        'MouthForm'
    ];

    // 检测张嘴参数
    for (const name of openCandidates) {
        try {
            if (coreModel.getParameterIndex(name) >= 0) {
                mouthCapability.openParam = name;
                break;
            }
        } catch (e) {}
    }

    // 检测嘴型参数
    for (const name of formCandidates) {
        try {
            if (coreModel.getParameterIndex(name) >= 0) {
                mouthCapability.formParam = name;
                break;
            }
        } catch (e) {}
    }

    // 确定模式
    if (mouthCapability.openParam && mouthCapability.formParam) {
        mouthCapability.mode = 'open_form';
    } else if (mouthCapability.openParam) {
        mouthCapability.mode = 'open_only';
    } else {
        mouthCapability.mode = 'unsupported';
    }

    console.log('[LipSync] 检测结果:', JSON.stringify(mouthCapability));
    return mouthCapability;
}
```

**调用时机：** 在 `loadModel()` 函数中，模型加载成功后、`pyBridge.onModelLoaded()` 之前调用。

#### 4.4.3 处理与现有 mouthSync 的冲突

现有的 `pixi-live2d-display` 库会在每一帧的 update 循环中用 `mouthSync()` 的返回值写入 LipSync 分组绑定的参数。如果我们同时通过 `setParameterValueById` 直接写参数，可能会被库的内部逻辑覆盖。

**解决办法：** 在说话期间，将 `mouthSync` 替换为我们控制的函数，让库的内部逻辑替我们写入 `openY`。`mouthForm` 则通过 `setParameterValueById` 直接写（库不会管这个参数）。

```javascript
let currentMouthOpenY = 0;
let isSpeakingNow = false;

function startRealLipSync() {
    if (!model || !model.internalModel) return;
    if (mouthCapability.mode === 'unsupported') return;

    const im = model.internalModel;
    const mm = im.motionManager;

    im.lipSync = true;
    mm.currentAudio = { ended: false, _fake: true };

    // 用我们的值替代 mouthSync
    mm.mouthSync = function() {
        return currentMouthOpenY;
    };

    isSpeakingNow = true;
}

function stopRealLipSync() {
    if (!model || !model.internalModel) return;
    const mm = model.internalModel.motionManager;

    if (mm.currentAudio && mm.currentAudio._fake) {
        mm.currentAudio = undefined;
    }

    mm.mouthSync = function() { return 0; };
    currentMouthOpenY = 0;
    isSpeakingNow = false;

    // 确保嘴巴闭合
    if (mouthCapability.formParam) {
        try {
            model.internalModel.coreModel.setParameterValueById(
                mouthCapability.formParam, 0
            );
        } catch (e) {}
    }
}

function updateMouth(openY, mouthForm) {
    // 更新全局变量，mouthSync() 会在下一帧读取
    currentMouthOpenY = openY;

    // mouthForm 直接写（不经过 mouthSync 机制）
    if (mouthCapability.formParam && model && model.internalModel) {
        try {
            model.internalModel.coreModel.setParameterValueById(
                mouthCapability.formParam, mouthForm
            );
        } catch (e) {}
    }
}
```

### 4.6 信号注意事项：线程安全

`LipSyncAnalyzer.feed()` 在 AudioPlayer 的播放线程中被调用，但 `visemeUpdated` 信号需要在主线程（Qt GUI 线程）中处理。

Qt 的信号槽机制默认使用 `AutoConnection`：如果发送者和接收者在不同线程，会自动变成 `QueuedConnection`，将调用放入主线程的事件队列。所以这里是线程安全的，不需要额外处理。

但需要注意：

- `feed()` 中的 `self.visemeUpdated.emit()` 会跨线程投递事件
- 主线程的 `_on_viseme_updated` -> `runJavaScript` 是在 GUI 线程执行
- 如果主线程繁忙，口型更新可能会有轻微延迟（通常 < 10ms，可接受）

## 六、效果预期

### 相比现在的提升

| 场景 | 现在 | 改造后 |
|------|------|--------|
| 说话时 | 嘴巴按固定节奏机械开合 | 跟随实际声音开合 |
| 停顿/逗号 | 嘴巴仍在动 | 自然闭合 |
| 大声/小声 | 无区别 | 大声张得大，小声张得小 |
| 摩擦音(s/sh/x) | 无区别 | 嘴型偏扁 |
| 圆唇音(o/u) | 无区别 | 嘴型偏圆（需模型有 ParamMouthForm） |
| 爆破音(b/p) | 无区别 | 短暂闭合后快速张开 |
| 不同模型 | 可能报错 | 自动检测，自动降级 |

### 仍然存在的限制

- 不是逐音素精确同步，而是"听起来像在说话"
- 连续元音之间的口型切换可能不够精确
- 对于极快语速，分析器可能跟不上每个音节
- `ParamMouthForm` 只能粗略分"扁/圆"，不能区分具体元音

## 七、后续优化方向（本次不做）

如果后续想进一步提升，可以考虑：

1. **引入 Rhubarb Lip Sync**：离线分析 TTS 生成的完整 wav，获得音素级时间轴。效果会有质的飞跃，但会增加外部依赖和延迟。

2. **TTS 换用支持时间戳的引擎**：例如 Azure TTS 的 `viseme_received` 事件可以直接返回口型 ID 和时间戳，完全跳过音频分析。

3. **给模型补 A/I/U/E/O 参数**：需要在 Live2D Cubism Editor 中打开模型工程文件（.cmo3），为嘴部网格新增五个形变参数。这是画师/模型师的工作。

4. **加入 numpy 加速**：如果 `array` 模块的性能不够，可以引入 numpy 做向量化计算，性能会提升 10~50 倍。但当前 25ms 间隔内的计算量很小，标准库够用。
